# Какой функционал планируется
- Аккаунты (Наличка, Карта, отложенные деньги)
- Категории (Выделение каждый на каждый месяц лимитов)
- Транзакции (Доходы, расходы, переводы, заметки)
- Статистика

# Описание подхода
Каждый месяц для каждой категории будет выделяться бюджет.
При создании обязательно выбирается аккаунт и категория в чей счёт она идёт
Все деньги, которые есть на балансе должны быть распланированны по категориям
Если по категории потрачено больше, чем выделялось - то это подсвечивает красным и нужно перераспределить
из других бюджетов деньги в эту категорию, либо этот долг перейдёт по этой категории на следующий месяц
Должна быть возможность скопировать настройки категорий с прошлого месяца
Если категория удаляется - то записи о ней пропадают и в других месяцах
Остатки денег передаются с прошлого месяца
По каждой категории нужно видеть сколько осталос, сколько выделено и сколько потратили

# API Endpoints — Roadmap (MVP)

### Авторизация
- [x] `POST   /api/v1/auth/register` → регистрация (username + password)
- [x] `POST   /api/v1/auth/login` → логин → устанавливает сессию
- [x] `POST   /api/v1/auth/logout` → логин → закрывает сессию

### Аккаунты
- [ ] `GET    /api/v1/accounts` → список аккаунтов + текущий баланс (вычисляется)
- [ ] `POST   /api/v1/accounts` → создать аккаунт + опционально начальный остаток (создаёт транзакцию)
- [ ] `PATCH  /api/v1/accounts/:id` → изменить название / порядок / заархивировать
- [ ] `DELETE /api/v1/accounts/:id` → удалить (или soft-delete)

### Категории
- [ ] `GET    /api/v1/categories` → все категории пользователя
- [ ] `POST   /api/v1/categories` → создать категорию (name + group?)
- [ ] `PATCH  /api/v1/categories/:id` → переименовать / изменить группу
- [ ] `DELETE /api/v1/categories/:id` → удалить (с проверкой использования)

### Транзакции
- [ ] `GET    /api/v1/transactions` → список с фильтрами: start/end/account_id/category_id
- [ ] `POST   /api/v1/transactions` → создать доход или расход (amount >0 = доход, <0 = расход)
- [ ] `PATCH  /api/v1/transactions/:id` → изменить + cleared/approved
- [ ] `DELETE /api/v1/transactions/:id` → удалить

### Бюджет и конверты (ядро!)
- [ ] `GET    /api/v1/budget/:year/:month` → полный экран месяца  
  → TBB, assigned, spent, available, carried_over для каждой категории
- [ ] `POST   /api/v1/budget/:year/:month/assign` → распределить деньги  
  { category_id, amount } — поддерживает отрицательные значения
- [ ] `POST   /api/v1/budget/:year/:month/quick-assign` → перенести значения assign категорий с прошлого месяца
